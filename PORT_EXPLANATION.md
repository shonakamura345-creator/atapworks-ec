# ポート番号について

## 現在の状況

### ポート3000
- ❌ **古いプロセスが実行中**（プロセスID: 41786）
- このプロセスは停止していないため、ポート3000が使用中
- `http://localhost:3000/` にアクセスすると、この古いプロセスに接続されます

### ポート3001
- ✅ **新しい開発サーバーが起動中**
- ポート3000が使用中のため、自動的にポート3001で起動
- `http://localhost:3001/` にアクセスすると、新しい開発サーバーに接続されます

---

## 正しいURL

新しい開発サーバーにアクセスするには：

```
http://localhost:3001/
```

---

## 対処方法

### オプション1: ポート3001を使用する（簡単）

新しい開発サーバーは既に起動しているので、そのまま使用：

1. ブラウザで `http://localhost:3001/` にアクセス
2. `stripe listen` もポート3001に変更：

```bash
stripe listen --forward-to localhost:3001/api/webhooks/stripe
```

### オプション2: 古いプロセスを停止してポート3000を使用

1. 古いプロセス（41786）を停止：

```bash
kill 41786
```

2. 現在の開発サーバーを停止（`Ctrl + C`）
3. 開発サーバーを再起動：

```bash
npm run dev
```

4. ポート3000で起動することを確認
5. `http://localhost:3000/` にアクセス

---

## 確認方法

### 開発サーバーのログで確認

開発サーバーのターミナルで、どのポートで起動しているか確認：

```
- Local:         http://localhost:3001  ← このポート番号を確認
```

このポート番号のURLにアクセスしてください。

---

## まとめ

- `http://localhost:3000/` → 古いプロセス（使用しない）
- `http://localhost:3001/` → 新しい開発サーバー（こちらを使用）

**推奨**: そのままポート3001を使用するのが簡単です。
